### Prerequisites

**Step 1: Check your Python version**

Explain: "The server requires Python 3.10 or newer. Python is a programming language — you're not writing any code, but you do need it installed to run the server."

Show the check command:
- macOS/Linux: `python3 --version`
- ⚠️ Windows callout: Use `python --version` instead (Windows installer adds `python`, not `python3`)

Expected output: `Python 3.10.x` or higher. If output shows 3.9 or lower, or "command not found":
- macOS: Install from python.org/downloads or via Homebrew (`brew install python`)
- Windows: Download from python.org/downloads — check "Add Python to PATH" during install
- Linux: `sudo apt install python3` (Ubuntu/Debian) or `sudo dnf install python3` (Fedora)

**Step 2: Get the repository**

Explain: "This is the folder containing the SAP knowledge content and the server files."

Primary path (no git required — simpler for most readers):
1. Go to github.com/BrandedTamarasu-glitch/SAPKnowledge
2. Click the green **Code** button → **Download ZIP**
3. Extract the ZIP to a folder you'll remember (e.g., `Documents/SAPKnowledge`)

Secondary path (if you have git):
```bash
git clone https://github.com/BrandedTamarasu-glitch/SAPKnowledge.git
```

**Step 3: Find your absolute path**

Explain: "The config files need the full path to your folder — not a shortcut. Think of it like a full postal address instead of 'my house'."

- Open Terminal (macOS/Linux) or PowerShell (Windows), navigate into the SAPKnowledge folder, then run:
  - macOS/Linux: `pwd` → example output: `/Users/yourname/Documents/SAPKnowledge`
  - Windows PowerShell: `Get-Location` → example output: `C:\Users\yourname\Documents\SAPKnowledge`

Tell the reader: "Copy this output — you'll need it in the steps below. Replace `/REPLACE/WITH/YOUR/PATH` in every example with this value."

**Step 4: Create the virtual environment**

Explain: "A virtual environment is an isolated copy of Python just for this project. It prevents the server's dependencies from conflicting with anything else on your computer."

- macOS/Linux:
  ```bash
  python3 -m venv .venv
  ```
- ⚠️ Windows callout: Use `python` instead of `python3`:
  ```powershell
  python -m venv .venv
  ```

**Step 5: Install the server's dependencies**

Explain: "This downloads the one library the server needs (fastmcp). It only runs once."

- macOS/Linux:
  ```bash
  .venv/bin/python -m pip install -r scripts/requirements.txt
  ```
- ⚠️ Windows callout:
  ```powershell
  .venv\Scripts\python.exe -m pip install -r scripts\requirements.txt
  ```

⚠️ Warning callout: "Do NOT run `pip install` without the `.venv/bin/python` prefix. Using the system pip would install into your global Python installation."

**Step 5 verification (smoke test):**

Explain: "Before touching any Claude config, confirm the server itself is runnable."

- macOS/Linux:
  ```bash
  .venv/bin/python -c "import fastmcp; print('OK')"
  ```
- Windows:
  ```powershell
  .venv\Scripts\python.exe -c "import fastmcp; print('OK')"
  ```
Expected output: `OK`

If you see an error mentioning Python version, your Python is older than 3.10 — revisit Step 1.

### Section 1: Claude Code

Explain: "Claude Code is the terminal-based Claude client. It auto-discovers MCP servers configured in a `.mcp.json` file at the project root — which this repo already provides."

**Step 6: Update .mcp.json with your path**

Explain: "The repo ships with a `.mcp.json` file that points to the author's computer. You need to update it to point to YOUR copy of the repo."

⚠️ Warning callout (prominent, placed BEFORE the template): "This is the most common setup failure. The file currently contains `/home/corye/Claude/SAPKnowledge/` — that path does not exist on your machine. You must replace it with your own path from Step 3."

Show what the file currently looks like (so they know what to find):
```json
{
  "mcpServers": {
    "sap-kb": {
      "command": "/home/corye/Claude/SAPKnowledge/.venv/bin/python",
      "args": ["/home/corye/Claude/SAPKnowledge/scripts/mcp_server.py"]
    }
  }
}
```

Show what they need to replace it with (macOS/Linux template):
```json
{
  "mcpServers": {
    "sap-kb": {
      "command": "/REPLACE/WITH/YOUR/PATH/.venv/bin/python",
      "args": ["/REPLACE/WITH/YOUR/PATH/scripts/mcp_server.py"]
    }
  }
}
```

⚠️ Windows callout (backslash warning — show after the macOS example):
"On Windows, use double backslashes (`\\`) in JSON, or use forward slashes — both work. Do NOT use single backslashes — they cause parse errors."

Windows template:
```json
{
  "mcpServers": {
    "sap-kb": {
      "command": "C:\\REPLACE\\WITH\\YOUR\\PATH\\.venv\\Scripts\\python.exe",
      "args": ["C:\\REPLACE\\WITH\\YOUR\\PATH\\scripts\\mcp_server.py"]
    }
  }
}
```

Instruction: Open `.mcp.json` in any text editor (Notepad on Windows, TextEdit on macOS, or nano in terminal). Replace both occurrences of the path.

**Step 7: Open Claude Code in the repo folder**

Instruction: Open Terminal/PowerShell, navigate into the SAPKnowledge folder, then run:
```bash
claude
```

⚠️ Note: "The first time you open Claude Code in this folder, it will ask permission to connect to the SAP KB server. Click **Allow**. (On some macOS versions this prompt may not appear — if so, the server loads automatically.)"

**Step 8: Verify Claude Code setup**

Two verification methods:

Method 1 — List servers from terminal (run before starting a Claude session):
```bash
claude mcp list
```
Expected: `sap-kb` appears in the list.

Method 2 — /mcp inside a session:
Type `/mcp` in the Claude Code prompt. Expected: `sap-kb` is listed as connected with 6 tools available.

If `sap-kb` shows as "disconnected": double-check the path in `.mcp.json` matches your actual folder location (Step 3).

**Step 9: Try your first query**

Ask Claude:
> What does T-code MIGO do?

Expected: Claude responds with a description of MIGO (Goods Movement transaction in MM), drawn from the SAP KB. If you see a real answer — setup is complete.

**Step 10: Open the Claude Desktop config file**

Easiest method — use the app's built-in editor:
Claude Desktop → Settings (gear icon) → Developer → **Edit Config**

**Step 11: Add the server entry**

Explain: "Add the `sap-kb` entry inside `mcpServers`. If `mcpServers` already exists, add the `sap-kb` block alongside any existing entries."

```json
{
  "mcpServers": {
    "sap-kb": {
      "command": "/REPLACE/WITH/YOUR/PATH/.venv/bin/python",
      "args": ["/REPLACE/WITH/YOUR/PATH/scripts/mcp_server.py"]
    }
  }
}
```

Replace `/REPLACE/WITH/YOUR/PATH` with your path from Step 3 (e.g., `/Users/yourname/Documents/SAPKnowledge`).

**Step 12: Quit and relaunch Claude Desktop**

⚠️ Warning callout: "Closing the window is NOT enough on macOS — the app stays running in the background. Use **Cmd+Q** to fully quit, then reopen Claude Desktop."

**Step 13: Verify Claude Desktop setup**

After relaunch: look for a hammer/tools icon in the bottom-right of the chat input area. Click it — `sap-kb` should appear with 6 tools listed.

Then ask:
> What does T-code MIGO do?

Expected: Claude Desktop responds with KB content. If you see a real answer — setup is complete.

If the server doesn't appear: check `~/Library/Logs/Claude/` for `mcp*.log` files — they show why the server failed to start.
