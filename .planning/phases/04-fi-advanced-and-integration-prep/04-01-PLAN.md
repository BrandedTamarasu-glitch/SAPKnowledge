# Financial Accounting — Account Determination

**Section 1: How Account Determination Works (OBYC Framework)**

Write a conceptual overview section explaining the full determination path:
```
Movement type (MIGO/MFBF) → OMJJ lookup → OBYC: Transaction key + Valuation class [+ Account modifier] → GL account
```

Explain each element:
- **Movement type**: The trigger (101 = GR against PO, 261 = GI to production, 551 = scrapping). Configured in MIGO or background via MFBF.
- **OMJJ**: Maps which OBYC transaction keys fire for each movement type. This is where the system looks up "movement type 101 triggers BSX, WRX, and optionally PRD."
- **Transaction keys**: The codes in OBYC that link to GL accounts. List the core keys: BSX (inventory/stock), WRX (GR/IR clearing), PRD (price differences), GBB (offsetting entries), KON (consignment payables).
- **Valuation class**: From the material master (Accounting 1 view, field BKLAS). Groups materials into categories for account determination. Example: 3000 = raw materials, 7920 = finished goods.
- **Account modifier**: Only THREE transaction keys support account modifiers: GBB, PRD, KON. BSX and WRX do NOT use account modifiers — they determine GL from valuation class alone. The account modifier further sub-classifies the posting (e.g., GBB with VBR = consumption, VNG = scrapping).

Include a subsection on **how to navigate OBYC**:
- **Reading current config (diagnostic):** T-code OBYC → enter transaction key (e.g., BSX) → system shows chart of accounts + valuation class + GL account assignment. Use this to answer "what account does BSX post to for valuation class 3000?"
- **Setting up during implementation:** T-code OBYC → select transaction key → New Entries → enter chart of accounts, valuation class, (account modifier if applicable), debit GL account, credit GL account. For GBB, also enter the account modifier (VBR, VNG, etc.).

**Section 2: MM Goods Receipt — Worked Example (Movement Type 101)**

Document all transaction keys that fire for movement type 101 (GR against PO):

| Transaction Key | Dr/Cr | What It Posts | Account Modifier | GL Account Determined By |
|----------------|-------|---------------|-----------------|-------------------------|
| BSX | Dr | Inventory / stock account | None | Valuation class only |
| WRX | Cr | GR/IR clearing account | None | Valuation class only |
| PRD | Dr or Cr | Price difference | Optional modifier | Valuation class (+ modifier if used) |

**Important note on PRD:** PRD fires for movement type 101 ONLY when the material has standard price control (price control indicator S in the material master Accounting 1 view). For MAP materials (price control indicator V), price differences are absorbed into BSX (the stock account value adjusts), and no separate PRD posting occurs.

**Section 4: VKOA Framework — SD Revenue Account Determination**

Write a framework-level introduction (Phase 8 owns the deep-dive). Cover:

- **What VKOA is:** The SD-side equivalent of OBYC. Maps billing document line items to revenue and COGS GL accounts. T-code VKOA.
- **KOFI vs KOFK:** KOFI = FI account determination (posts to revenue GL accounts). KOFK = CO account determination (posts to CO-PA profitability objects). Phase 4 focuses on KOFI only; KOFK is CO integration deferred to Phase 10.
- **How VKOA determines the GL account (KOFI access sequence KOFI00):** The system searches through 5 access levels from most specific to least specific. First match wins.

| Access Level | Fields Used | Meaning |
|-------------|------------|---------|
| 1 | Chart of Accounts + Sales Org + Cust AAG + Mat AAG + Account Key | Most specific match |
| 2 | Chart of Accounts + Sales Org + Cust AAG + Account Key | No material group |
| 3 | Chart of Accounts + Sales Org + Mat AAG + Account Key | No customer group |
| 4 | Chart of Accounts + Cust AAG + Mat AAG + Account Key | No sales org |
| 5 | Chart of Accounts + Account Key | Catch-all fallback |

- **Key fields driving VKOA:**
  - **KTGRD** (Customer Account Assignment Group): Stored in KNVV (customer sales area data), Billing Documents tab. Classifies customers (e.g., 01 = domestic revenue, 02 = export revenue, 03 = intercompany). See modules/fi/master-data.md KNVV section.
  - **Material Account Assignment Group**: Stored in the material master, Sales Org 2 view (field "Acct assmt grp material"). Classifies materials for revenue determination.
  - **Account Key**: From the pricing procedure (V/08). Standard keys: ERL = revenue, ERS = sales deductions/discounts, ERF = freight revenue, MWS = output tax.

- **Scope boundary:** Phase 4 establishes this framework. Phase 8 (SD Advanced) owns the full VKOA deep-dive including condition tables, access sequences, account assignment groups configuration, and worked billing-to-GL examples.

**Section 5: FI-Only Automatic Postings**

Cover automatic postings that originate within FI (not from MM or SD integration):

**5a. Foreign Currency Valuation — F.05 / OBA1**

- F.05 runs month-end revaluation of foreign currency open items and balances.
- Configuration in OBA1 defines which gain/loss GL accounts receive the valuation differences.
- Two key groupings:
  - **KDB** — Non-open-item GL accounts (balance sheet FX accounts). The GL master record (FS00) has an "Exchange rate difference key" field linking to OBA1/KDB config.
  - **KDF** — Open items: vendor/customer open items AND open-item GL accounts. Enter the reconciliation account, assign gain/loss GL accounts for that grouping.
- **IMG Path:** SPRO → Financial Accounting (New) → General Ledger Accounting (New) → Periodic Processing → Valuate → Define Accounts for Foreign Currency Valuation

**5b. Cash Discount GL Account Configuration — OBXU / OBXI**

**CORRECTION from CONTEXT.md:** Cash discount GL accounts are NOT configured in OBB8. OBB8 defines payment terms (days, percentages) only.

- **OBXU** — GL account for cash discount received (AP side: vendor grants a discount when company pays early)
- **OBXI** — GL account for cash discount paid (AR side: company grants a discount to customer for early payment)
- These are accessed via SPRO → Financial Accounting (New) → Accounts Receivable and Accounts Payable → Business Transactions → Outgoing Payments/Incoming Payments → Define Accounts for Cash Discount Taken/Granted

**5c. Clearing Account Mechanics**

Explain the role of clearing accounts in FI automatic postings:
- **GR/IR clearing (191100-type)**: Debited by MIRO (invoice verification), credited by MIGO (goods receipt via WRX). F.13 automatic clearing matches GR and IR postings.
- **Asset clearing accounts**: Used when F-91 posts asset acquisition to a clearing account instead of directly to vendor. Cleared when vendor invoice arrives.
- **Payroll clearing accounts**: (Mentioned for completeness; HR integration not in scope.)
- The clearing account must be flagged as "open item managed" in FS00 (OI indicator) for F.13 automatic clearing to function. If the GR/IR account is not open-item managed, F.13 will not clear it — this is a common implementation pitfall.
