## Accounts Payable (AP)

AP T-codes (15) — use RESEARCH.md verified data:
1. FB60 — Enter Vendor Invoice (Enjoy). Menu: Accounting → Financial Accounting → Accounts Payable → Document Entry → Invoice. Usage: Preferred for simple vendor invoices; park or post immediately.
2. F-43 — Enter Vendor Invoice (classic). Menu: Accounting → Financial Accounting → Accounts Payable → Document Entry → Invoice - General. Note: Requires posting keys (31 = vendor credit line; 40 = debit GL line).
3. FB65 — Enter Vendor Credit Memo (Enjoy). Menu: Accounting → Financial Accounting → Accounts Payable → Document Entry → Credit Memo. Note: Symmetric to FB60; use for vendor credit memos.
4. F-41 — Enter Vendor Credit Memo (classic). Menu: Accounting → Financial Accounting → Accounts Payable → Document Entry → Credit Memo - General. Gotcha: F-41 = VENDOR credit memo. F-27 = CUSTOMER credit memo. Do not mix.
5. MIRO — Invoice Verification (from MM). Menu: Logistics → Materials Management → Logistics Invoice Verification → Document Entry → Enter Invoice. Note: MM-side T-code used for PO-based invoices. Generates FI document automatically via MM-FI integration; does not require FI posting knowledge.
6. F-53 — Post Outgoing Payment (manual). Menu: Accounting → Financial Accounting → Accounts Payable → Document Entry → Outgoing Payment → Post. Usage: Manual vendor payment without the automatic payment program; use for one-off payments.
7. F-58 — Payment with Printout. Menu: Accounting → Financial Accounting → Accounts Payable → Document Entry → Outgoing Payment → Post + Print Forms. Usage: Manual check printing combined with the payment posting.
8. F110 — Automatic Payment Program (APP). Menu: Accounting → Financial Accounting → Accounts Payable → Periodic Processing → Payments. Usage: Mass payment run. Full execution sequence: (1) FBZP pre-configuration, (2) F110 Parameters tab, (3) Proposal run, (4) Edit proposal, (5) Payment run, (6) DME/printout, (7) FBL1N reconciliation. Gotcha: Deleting a proposal and re-running with the same run date + identification picks up newly due items — use a new identification for a fresh run.
9. FBL1N — Vendor Line Items. Menu: Accounting → Financial Accounting → Accounts Payable → Account → Display/Change Line Items. Usage: View open, cleared, or all items for a vendor account. Reads BSIK (open) and BSAK (cleared).
10. FK01 — Create Vendor (accounting view only). Menu: Accounting → Financial Accounting → Accounts Payable → Master Records → Create. Usage: FI-only vendor; no purchasing data. Use XK01 in MM+FI environments.
11. FK02 — Change Vendor (accounting view). Menu: Accounting → Financial Accounting → Accounts Payable → Master Records → Change.
12. FK03 — Display Vendor (accounting view). Menu: Accounting → Financial Accounting → Accounts Payable → Master Records → Display.
13. XK01 — Create Vendor (all views, central). Menu: Logistics → Materials Management → Purchasing → Master Data → Vendor → Central → Create. Usage: Creates LFA1 (general) + LFB1 (company code) + LFM1 (purchasing org) in one transaction. Preferred in environments with both MM and FI.
14. F150 — Dunning Run. Menu: Accounting → Financial Accounting → Accounts Receivable → Periodic Processing → Dunning. Usage: Executes dunning program SAPF150; generates dunning notices for overdue customer/vendor items.
15. S_ALR_87012082 — Vendor Balances in Local Currency. Menu: Accounting → Financial Accounting → Accounts Payable → Information System → Reports for Accounts Payable Accounting → Vendor Balances → Vendor Balances in Local Currency. CRITICAL NOTE: S_ALR_87012082 = vendor balances. S_ALR_87012284 = Financial Statements (Balance Sheet/P&L). These are frequently confused.

2. F-22 — Enter Customer Invoice (classic). Menu: Accounting → Financial Accounting → Accounts Receivable → Document Entry → Invoice - General. Note: Requires posting keys (01 = customer debit; 50 = credit GL).
3. FB75 — Enter Customer Credit Memo (Enjoy). Menu: Accounting → Financial Accounting → Accounts Receivable → Document Entry → Credit Memo.
4. F-27 — Enter Customer Credit Memo (classic). Menu: Accounting → Financial Accounting → Accounts Receivable → Document Entry → Credit Memo - General. Gotcha: F-27 = CUSTOMER credit memo. F-41 = VENDOR credit memo.
5. F-28 — Post Incoming Payment. Menu: Accounting → Financial Accounting → Accounts Receivable → Document Entry → Incoming Payments. Usage: Manual customer payment receipt posting with clearing.
6. F-32 — Clear Customer Account. Menu: Accounting → Financial Accounting → Accounts Receivable → Account → Clear. Usage: Manual clearing of customer open items.
7. FBL5N — Customer Line Items. Menu: Accounting → Financial Accounting → Accounts Receivable → Account → Display/Change Line Items. Usage: View open, cleared, or all items for a customer account. Reads BSID (open) and BSAD (cleared).
8. FD01 — Create Customer (accounting view only). Menu: Accounting → Financial Accounting → Accounts Receivable → Master Records → Create. Usage: FI-only customer; no sales data. Use XD01 in SD+FI environments.
9. FD02 — Change Customer (accounting view). Menu: Accounting → Financial Accounting → Accounts Receivable → Master Records → Change.
10. FD03 — Display Customer (accounting view). Menu: Accounting → Financial Accounting → Accounts Receivable → Master Records → Display.
11. XD01 — Create Customer (all views, central). Menu: Logistics → Sales and Distribution → Master Data → Business Partners → Customers → Create → Complete. Usage: Creates KNA1 + KNB1 + KNVV in one transaction. Preferred in SD+FI environments.
12. FD32 — Customer Credit Limit. Menu: Accounting → Financial Accounting → Accounts Receivable → Credit Management → Credit Management → Change. Usage: Set or display credit limit, credit control area assignment, and credit exposure for a customer.
13. S_ALR_87012168 — Due Date Analysis for Open Items. Menu: Accounting → Financial Accounting → Accounts Receivable → Information System → Reports → Customer Balances → Due Date Analysis. Usage: Aged receivables / overdue analysis by customer.

## Asset Accounting (AA)

AA T-codes (12):
1. AS01 — Create Asset Master Record. Menu: Accounting → Financial Accounting → Fixed Assets → Asset → Create → Asset. Usage: Creates ANLA record (general segment) and associated ANLB (depreciation terms) for each depreciation area. Asset class selection drives number range and depreciation defaults.
2. AS02 — Change Asset Master Record. Menu: Accounting → Financial Accounting → Fixed Assets → Asset → Change.
3. AS03 — Display Asset Master Record. Menu: Accounting → Financial Accounting → Fixed Assets → Asset → Display.
4. ABUMN — Asset Transfer (within company code). Menu: Accounting → Financial Accounting → Fixed Assets → Posting → Transfer → Transfer Within Company Code. Usage: Transfers between asset classes, cost centers, or business areas. Creates both retirement and acquisition postings.
5. F-90 — Asset Acquisition from Vendor with Purchase Order. Menu: Accounting → Financial Accounting → Fixed Assets → Posting → Acquisition → External Acquisition → With Vendor. Usage: Asset purchase with vendor clearing; creates both asset acquisition document and vendor open item.
6. F-91 — Asset Acquisition to Clearing Account. Menu: Accounting → Financial Accounting → Fixed Assets → Posting → Acquisition → External Acquisition → Acquis. w/Autom. Offsetting Entry. Usage: Acquires asset to an interim/clearing account when vendor clearing happens separately.
7. F-92 — Asset Retirement with Customer. Menu: Accounting → Financial Accounting → Fixed Assets → Posting → Retirement → Retirement w/Revenue → With Customer. Usage: Asset sale generating customer receivable and gain/loss on disposal.
8. ABAVN — Asset Retirement by Scrapping. Menu: Accounting → Financial Accounting → Fixed Assets → Posting → Retirement → Asset Retirement by Scrapping. Usage: Write-off without sale revenue; posts loss on scrapping.
9. AFAB — Depreciation Run. Menu: Accounting → Financial Accounting → Fixed Assets → Periodic Processing → Depreciation Run → Execute. Usage: Posts periodic depreciation for all assets. Can run in test mode first. Gotcha: Must complete AFAB fully before running AJAB (year-end close) — AJAB checks all depreciation is posted.
10. AJRW — Asset Fiscal Year Change. Menu: Accounting → Financial Accounting → Fixed Assets → Periodic Processing → Fiscal Year Change. Usage: Opens the new fiscal year for asset accounting; must run before posting in the new year.
11. AJAB — Asset Year-End Closing. Menu: Accounting → Financial Accounting → Fixed Assets → Periodic Processing → Year-End Closing → Execute. Usage: Closes the current asset fiscal year. Gotcha: Runs AFTER AFAB (depreciation) is 100% complete and AFTER AJRW has opened the new year. If depreciation is not fully posted, AJAB will fail.
12. AW01N — Asset Explorer. Menu: Accounting → Financial Accounting → Fixed Assets → Asset → Asset Explorer. Usage: Single-asset drilldown showing planned vs posted depreciation values per depreciation area. Most useful tool for diagnosing asset depreciation issues.

## Period-End and Cross-Functional Reporting

5 T-codes covering period-end and financial reporting:
1. F.01 — Financial Statements. Menu: Accounting → Financial Accounting → General Ledger → Information System → General Ledger Reports → Balance Sheet / Profit+Loss Statement / Cash Flow → General → Actual/Actual Comparisons → Financial Statements. Usage: Runs program RFBILA00. Requires Financial Statement Version (FSV) configured in OB58. Same underlying program as S_ALR_87012284.
2. S_ALR_87012284 — Financial Statements (Balance Sheet/P&L). Menu: Accounting → Financial Accounting → General Ledger → Information System → Reports for General Ledger Accounting → Balance Sheet / Profit+Loss Statements → General → Balance Sheet / P+L Statement. Note: Parameter variant for RFBILA00. CRITICAL: This is FINANCIAL STATEMENTS, not vendor balances. Vendor balances = S_ALR_87012082.
3. S_ALR_87012277 — GL Account Balances (Trial Balance). Menu: Accounting → Financial Accounting → General Ledger → Information System → Reports for General Ledger Accounting → General → G/L Account Balances. Usage: Trial balance by GL account; shows period and cumulative balances.
4. FBL3N — GL Account Line Items (Classic GL). Menu: Accounting → Financial Accounting → General Ledger → Account → Display/Change Line Items. Usage: Line item display for GL accounts; reads BSIS (open) and BSAS (cleared). Note: Works in New GL environments but does not show New GL-specific fields (segment, profit center per line). Use FAGLL03 for New GL line item display.
5. OBY6 — Company Code Global Settings Display. Menu: IMG path — SPRO → Enterprise Structure → Definition → Financial Accounting → Edit, Copy, Delete, Check Company Code. Usage: Displays assigned fiscal year variant, chart of accounts, currency, and posting period variant for a company code.
